It's actually a well-known pattern called multi-hypothesis tracking: maintain N parallel candidate paths (each with their own running x and history of N recent positions), score each by how "linelike" it has been over its history (low variance, support density, persistence), and at any row prefer the candidate path with the best history score. When a candidate drops below threshold (no continuation), kill it. New candidates can be spawned from divergent candidates in any row.Let me prototype that properly.
